Capacity

First-time setup

Work down the steps. Only the last one talks to the panel.

1
Add your Surfshark key
The WireGuard private key from your Surfshark account.
2
Reserve slots
Reserve more than you need — adding slots later is the only thing that requires binding again. Empty slots cost nothing.
3
Fill slots with countries
Each slot gets a different country and is pinned to it. Fine-tune afterwards on the Slots tab.
4
Bind ports to the panel
Run once. After this, key and country changes never touch the panel again.
A slot's port and country are both fixed. Key rotations and server failures cannot change either one.
Rotating a key does not require re-injection. Affected tunnels restart on their existing ports; inbounds, UUIDs and user subscriptions are untouched.

Keys

Scheduled rotation

Reshuffles which key serves which slot on a timer, so no keypair holds the same long-lived set of sessions. Ports, countries and user configs are untouched. Needs at least two keys to do anything.

Add another key

Several keypairs can live on one Surfshark account — generate them in the account's WireGuard section. If one key gets culled, rotate only that one; slots on other keys aren't even restarted.

Bulk import

For large deployments. Paste one key per line, label: key, key,address, or whole [Interface] blocks. Duplicates are skipped.

This layout is derived from core/config.py, so it is identical on every machine — this server now, or a rebuilt one later.
